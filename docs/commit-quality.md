# Commit Quality

Use Conventional Commit style with a clear scope when possible.

## Recommended

- `feat(engine): add collision pipeline`
- `fix(ui): resolve inventory overlay`
- `refactor(audio): simplify mixer`
- `docs: update rendering architecture`
- `ci: add repository statistics workflow`

## Avoid

- `Update`
- `Fix`
- `Changes`
- `Stuff`

## Commit Checklist

- Explain what changed.
- Include the affected system in the scope.
- Keep commits focused.
- Update `docs/CHANGELOG.md` for meaningful changes.
- Add an AI activity entry for AI-generated commits.
