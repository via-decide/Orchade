# Repository Map

## Scope
This map records static evidence from `src/`, `data/`, `gameplay/`, and existing `docs/` as of the repository state that produced this file.

## Top-level areas
| Path | Evidence | Current role | Notes |
|---|---|---|---|
| `src/` | `src/App.tsx`, `src/types.ts`, `src/constants.ts`, `src/components/*`, `src/simulation/*` | Runnable legacy React/Vite orchard app plus an emerging simulation layer. | `README.md` says `src/` remains the legacy runnable shell during migration. |
| `data/` | JSON definitions for crops, items, weather, tools, buildings, NPCs, and quests. | Data-pack seed content. | Files share `schemaVersion` and `entries`, matching the `DataPack<T>` shape in `src/simulation/types.ts`. |
| `gameplay/` | Ten feature capsules in `gameplay/module-manifest.json`. | Target architecture for isolated gameplay ownership. | Capsules are scaffolded with README, API, state, UI, TODO, tests, AI, and internals. |
| `docs/` | Dashboard, health, metrics, events, ADRs, graphs, gameplay architecture. | Engineering memory and architecture evidence. | Generated by `tools/generate-engineering-docs.mjs` according to `README.md`. |
| `tools/` | `tools/generate-engineering-docs.mjs` | Repository intelligence automation. | Supports `npm run docs:generate`. |

## Runtime feature evidence
| Feature | Evidence | Observed implementation state |
|---|---|---|
| Farming loop | Plant state includes root strength, water, nutrients, stress, pests, stage, harvestability. Actions in `src/App.tsx` include research, water, fertilize, pesticide, and harvest. | Playable in the legacy shell; capsule extraction remains incomplete. |
| Weather | `WEATHER_TYPES`, random forecast, climate control, weather visual effects, and `data/weather/weather-definitions.json`. | Playable UI logic plus data-pack seed; deeper system integration planned. |
| Economy | Credits, Data Seeds, shop items, upgrade costs, seed costs, transfers, rankings. | Playable monetization/progression loop; economy capsule is scaffolded. |
| Collection | `harvestedTypes` unlocks encyclopedia entries. | Strong retention foundation but currently tied to harvest events only. |
| World | Multiple orchards, unlock costs, active orchard, day progression, world system stubs. | Basic world expansion exists; systemic locations are planned. |
| Simulation platform | Event bus, ordered systems, boot/tick/autosave phases. | Non-UI simulation architecture exists but is not fully connected to the React shell. |

## Data-pack inventory
| Data file | Evidence | Expansion opportunity |
|---|---|---|
| `data/crops/crop-definitions.json` | One crop with seasons, stages, water, nutrients, harvest, disease, and fertilizer effects. | Add species variety and align with runtime seed list. |
| `data/items/item-definitions.json` | Resource, seed, and tool examples with stack and weight metadata. | Back an actual inventory UI and crafting loop. |
| `data/weather/weather-definitions.json` | Rain, storm, and fog with movement, visibility, lighting, crop water, and audio fields. | Extend to clear/heatwave and drive rendering/audio/system effects. |
| `data/quests/quest-definitions.json` | First harvest quest with trigger and reward. | Add tutorial, daily, seasonal, and NPC quests. |
| `data/npcs/npc-archetypes.json` | Villager, animal, merchant archetypes with schedule/needs/memory/task traits. | Implement NPC routines and social economy. |

## Architecture risks and opportunities
| Suggestion | Why it improves the game | Dev effort | Technical risk | Gameplay impact | Replayability impact | Retention impact | Architecture impact |
|---|---|---:|---:|---:|---:|---:|---:|
| Connect the `src/simulation` orchestrator to the live React state through a thin adapter. | Converts existing static simulation systems into visible day/weather/autosave behavior and creates one place for future systems to run. | L | M | H | M | H | H |
| Promote `data/*` JSON into runtime-loaded catalogs. | Lets designers add crops, weather, items, quests, and NPCs without editing monolithic React code. | M | M | H | H | H | H |
| Move farming logic from `src/App.tsx` into `gameplay/farming`. | Makes the core loop testable, safer to balance, and easier for AI contributors to modify. | M | L | M | M | M | H |
| Add capsule tests for each public API before feature expansion. | Prevents regressions while adding content and systems. | M | L | M | M | M | H |
| Update docs index to include product-design docs. | Keeps roadmaps discoverable beside engineering docs. | S | L | L | L | M | M |

## Scale
Effort, risk, and impact use `S`, `M`, `L`, and `H` as relative planning estimates.
